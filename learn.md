## 1. 视觉基础模型：从 ImageNet 到自监督学习

### 1.1 什么是视觉基础模型

视觉基础模型（Vision Foundation Model）是在海量图像数据上预训练的大型神经网络，其输出的特征表示可以作为"通用视觉语言"直接用于分类、分割、检索、匹配等多种下游任务，无需针对每个任务从头训练。

与传统的 ImageNet 监督预训练不同，现代视觉基础模型采用**自监督学习**（Self-Supervised Learning），不依赖人工标注。

### 1.2 DINO 系列的发展

| 版本 | 年份 | 核心创新 | 数据规模 |
|------|------|---------|---------|
| DINO | 2021 | 自蒸馏 + 动量编码器，无需负样本 | ImageNet-1K |
| DINOv2 | 2023 | iBOT 联合训练 + KoLeo 正则化 | LVD-142M（1.42 亿张） |
| **DINOv3** | 2024 | + Register Tokens + 优化训练配方 | LVD-1689M（~1.7 亿张） |

**核心思想**：DINO 系列使用"学生-教师"（Student-Teacher）框架。教师模型的参数是学生模型参数的指数移动平均（EMA），学生学习教师的输出。两个模型看同一图像的不同 augmented 版本（crop、color jitter 等），学生必须预测教师在同一图像上的特征分布。

关键洞察：**不需要负样本**。传统的对比学习（如 SimCLR）需要大量负样本来防止模型坍塌到平凡解（所有输出相同），DINO 通过 centering + sharpening 操作在 softmax 后平衡输出分布，自然避免了坍缩。

### 1.3 为什么 DINO 特征适合下游任务

DINO 的特征具有显著的**语义平滑性**：
- "门把手"的 patch 和另一个"门把手"的 patch 天然相似
- "白墙"的 patch 和另一边"白墙"的 patch 天然相似
- 这种相似性不是通过标签学习的，而是因为自监督目标要求模型理解图像的内在结构

这种语义平滑性让 DINO 在 Zero-Shot 场景下就能进行图像匹配——同一个 3D 物体从不同视角看，其 patch 在语义空间中是相近的。

## 2. DINOv3 架构深度解析

### 2.1 Vision Transformer (ViT) 基础

DINOv3 使用 ViT-L/16 架构。对于一个 H×W 的输入图像：

```
输入图像 (448×448×3)
    ↓ 切分为 16×16 的 patch → 28×28 = 784 个 patches
    ↓ Linear Projection (16×16×3 → 1024)
    ↓ 加 position embedding + 4 个 register tokens
    ↓ 24 层 Transformer（每层：Self-Attention + MLP）
    ↓ 输出：每个 patch 一个 1024-dim 特征向量
```

**关键参数**：

| 参数 | 值 | 说明 |
|------|-----|------|
| 层数 | 24 | Transformer block 数量 |
| 隐藏维度 | 1024 | 每层输出的特征维度 |
| 注意力头数 | 16 | Multi-head attention 的头数 |
| Patch 大小 | 16×16 | 每个 patch 覆盖的像素区域 |
| MLP 扩展比 | 4 | FFN 内部维度 = 1024×4 = 4096 |
| 总参数量 | ~300M | 不含 head |
| 有效 patch 数 | 784 | 448/16 × 448/16 |

### 2.2 RoPE（旋转位置编码）

DINOv3 使用 2D RoPE（Rotary Position Embedding）而非传统的可学习位置编码。

**原理**：RoPE 通过对 Query 和 Key 向量施加旋转变换来编码位置信息。对于位置 m 和 n，旋转角度为 (m-n)·θ，使得 attention 分数天然包含相对位置信息。

**2D 扩展**：DINOv3 为 patch 的 (row, col) 各分配一组频率，分别编码行位置和列位置。两组旋转拼在一起形成完整的 2D 位置编码。

**优势**：
- 天然的外推能力（训练 224×224，可直接用到 448×448）
- 相对位置信息直接编码在 attention 中
- 不需要学习参数

### 2.3 LayerScale

每个 Transformer 子层（Attention 和 MLP）的输出会乘以一个可学习的缩放因子（初始化为一个很小的值，如 ε=1e-5）：

```
output = x + LayerScale(Sublayer(x))
       = x + diag(λ₁, λ₂, ..., λd) · Sublayer(x)
```

**作用**：
- 训练初期，λ≈0 意味着每层几乎输出恒等映射 → 模型从接近恒等的状态开始学习
- 改善了深层 ViT 的训练稳定性
- 每个通道有独立的缩放因子，允许模型逐步"开启"不同通道

### 2.4 Register Tokens

DINOv3 在 patch tokens 之外额外添加了 4 个 register tokens（存储令牌），它们是可学习的向量，与 patch tokens 一起经过所有 Transformer 层。

**动机**：研究发现 ViT 的特征图中会出现"伪影"（高范数的异常 patch），这些伪影干扰下游任务。Register tokens 作为"废纸篓"——模型把全局信息（如整张图的亮度、色调）存入 register tokens，让 patch tokens 专注于局部信息。

**在我们的项目中的处理**：提取特征时，register tokens 被丢弃，只保留 784 个 patch tokens。

### 2.5 特征提取流程

```
图像 (H×W×3)
    ↓ resize → 448×448
    ↓ 切分 → 28×28 patches
    ↓ PatchEmbed → 784 × 1024
    ↓ + 4 register tokens → 788 × 1024
    ↓ + RoPE 2D position encoding
    ↓ 24× Transformer Block (Self-Attn + MLP + LayerScale)
    ↓ 丢弃 register tokens → 784 × 1024
    ↓ L2 Normalize（每个向量归一化到单位球面）
    ↓ 输出：784 个 1024-dim 归一化特征
```

## 3. 局部特征匹配：问题定义与技术栈

### 3.1 问题定义

**输入**：两张有重叠区域的图像 (A, B)，从不同视角拍摄同一场景。

**输出**：像素级的对应关系——图 A 的 (x₁, y₁) 对应图 B 的 (x₂, y₂)。

**应用**：三维重建（SfM）、视觉定位、SLAM、AR

### 3.2 MNN 匹配（Mutual Nearest Neighbor）

给定两组特征（图 A 的 784 个特征向量和图 B 的 784 个特征向量）：

1. **前向匹配**：对 A 的每个 patch i，在 B 中找余弦相似度最高的 patch j
2. **反向匹配**：对 B 的每个 patch j，在 A 中找余弦相似度最高的 patch k
3. **双向互检**：只有 i = k（即 A→B 和 B→A 的最近邻一致）才算有效匹配

**为什么用 MNN 而非单向 NN**：单向最近邻会产生大量错误匹配（尤其是有重复纹理或纹理缺失区域时）。MNN 的双向一致性约束显著提高了匹配精度。

**余弦相似度**：

```
cos_sim(a, b) = a·b / (||a||·||b||)
```

由于 DINOv3 特征通常是 L2 归一化的，余弦相似度退化为点积：`cos_sim = a·b`。

### 3.3 Essential Matrix 与对极几何

**对极几何（Epipolar Geometry）**描述了同一 3D 点在两个相机成像平面上的投影之间的几何关系。

给定一对匹配点 (x₁, x₂)（齐次坐标），它们满足：

```
x₂ᵀ · E · x₁ = 0
```

其中 **E**（Essential Matrix，本质矩阵，3×3）编码了两个相机之间的相对旋转 R 和平移 t：

```
E = [t]× · R
```

**对极线**：给定图 A 中的点 x₁，它在图 B 中的对应点 x₂ 必定位于对极线 l₂ = E·x₁ 上。这个约束是过滤错误匹配的核心工具。

**对极距离**：x₂ 到对极线 l₂ 的距离，用于衡量匹配是否满足几何约束：

```
epipolar_error = |x₂ᵀ · E · x₁| / sqrt(l₂[0]² + l₂[1]²)
```

Precision 指标定义为：对极误差 < 5×10⁻⁴ 的匹配点占比。

### 3.4 Essential Matrix 估计

从有噪声的匹配点对中估计 E：

1. **五点算法（Nistér, 2004）**：最少 5 对匹配点即可求解 E（E 有 5 个自由度：3 个旋转 + 2 个平移方向（平移尺度不可恢复））
2. **USAC（Universal RANSAC）**：鲁棒估计框架，处理大量外点：
   - 随机采样最小的点集（5 对）
   - 估计 E
   - 统计 inlier 数量（对极误差 < 阈值的匹配点）
   - 在 inlier 上重新估计 E
   - 重复直到找到最优 E

USAC 相比传统 RANSAC 的改进：自适应采样、预检验、局部优化——更少迭代达到同样精度。

### 3.5 相机位姿恢复

从 Essential Matrix E 恢复相对位姿 (R, t)：

1. **SVD 分解**：E = U · Σ · Vᵀ
2. 通过 Σ 修正得到两种可能的 R 和两种可能的 t
3. 组合出 4 个可能的 (R, t)
4. **Cheirality Check**：用三角化判断哪个解的 3D 点在两个相机前方 → 唯一确定正确解

### 3.6 评估指标

**AUC@k°**（Area Under the Curve at k degrees）：统计位姿角度误差 < k° 的累积分布面积。

- 对每个测试对，计算估计位姿与真值位姿之间的角度误差
- 角度误差 = max(旋转误差, 平移方向误差)，单位：度
- 统计误差 < k° 的图像对占比
- AUC@5: 最严格（对应高精度 AR/机器人抓取）
- AUC@20: 较宽松（对应粗略定位）

**Precision**：匹配点中满足对极几何约束的比例（对极误差 < 5×10⁻⁴）。衡量的是"单个匹配的质量"而非"整体位姿的精度"。

**关键洞察**：Precision 高不一定 AUC 高。如果高 Precision 的匹配点集中分布在图像的某一小块区域，位姿估计会因为缺乏空间分布而退化。AUC 要求匹配点在空间上分布均匀。

## 4. 微调技术：LoRA 与对比学习

### 4.1 为什么需要参数高效微调

DINOv3 有 ~300M 参数。全量微调面临三个问题：
1. **显存需求巨大**：300M 参数的优化器状态（Adam: 参数 + 动量 + 方差）需要 ~3.6GB
2. **灾难性遗忘**：少量数据上全量微调会覆盖预训练知识
3. **过拟合风险**：相比 1.42 亿张预训练图片，几千张微调图片几乎必然导致过拟合

### 4.2 LoRA（Low-Rank Adaptation）

**核心思想**：不直接修改原始权重 W，而是注入一个低秩增量 ΔW = B·A：

```
h = W·x + ΔW·x = W·x + B·A·x
```

其中：
- W ∈ R^{d×d}（冻结，不训练）
- A ∈ R^{r×d}（可训练，Kaiming 随机初始化）
- B ∈ R^{d×r}（可训练，零初始化）
- r << d（秩，通常 r=4 或 8）

**关键特性**：
1. **B=0 初始化**：训练开始时 ΔW = 0，模型输出完全等于预训练模型。这保证了"最差情况就是退回 Zero-Shot"。
2. **参数高效**：r=4 时，每个注意力矩阵只增加 4×1024×2 = 8192 个参数
3. **即插即用**：可以注入任意 nn.Linear 层

**在我们的项目中的配置**：
- 在 ViT 的 24 层中，每层的 Q、K、V、Proj 四个注意力矩阵都注入 LoRA
- 每个矩阵的原始大小：1024×1024
- LoRA 参数量：24 层 × 4 矩阵 × 1024×4×2 = ~0.39M（约占 backbone 的 0.13%）

### 4.3 对比学习（Contrastive Learning）

对比学习的核心操作：**拉近正样本对，推开负样本**。

在 DINOv3 特征空间中，每个特征向量都在 1024 维单位超球面上（因为 L2 归一化）。余弦相似度被用作"距离"度量。

### 4.4 InfoNCE 损失

InfoNCE（Noise Contrastive Estimation）是最广泛使用的对比学习损失函数：

```
L_i = -log[ exp(s_i⁺/τ) / ( exp(s_i⁺/τ) + Σ_j exp(s_j⁻/τ) ) ]
```

其中：
- s_i⁺：正样本对的余弦相似度
- s_j⁻：第 j 个负样本对的余弦相似度
- τ：温度参数（temperature），控制分布的"尖锐"程度
- Σ_j 遍历所有 K 个负样本

**温度 τ 的关键作用**：τ 越小，softmax 越接近 one-hot（硬竞争）；τ 越大，分布越平滑（软竞争）。SimCLR 和 DINO 使用 τ=0.07（非常小）。

**当 InfoNCE 用于密集匹配时的致命问题**：

τ=0.07 意味着余弦相似度被放大了 1/0.07 ≈ 14.3 倍。原本在 [0, 1] 范围的相似度被放大到 [0, 14.3]，softmax 几乎退化为 argmax——只有"最相似的"那个负样本被推开，其他负样本的梯度近乎为零。

在密集匹配中，同一图像中有数百个语义相似的 patch（比如一面白墙上的不同位置）。InfoNCE 强制模型从中选出一个赢家而推开所有其他——这对需要保持语义连续性的密集特征来说是灾难性的。

**数学上**：当模型完全坍塌，所有特征退化为均匀随机分布时，InfoNCE 的期望值为：
```
L = ln(K+1) ≈ ln(129) ≈ 4.8598
```
（K=128 个负样本 + 1 个正样本）

### 4.5 Margin-based 损失

作为 InfoNCE 的替代，margin-based 损失不使用 softmax 竞争：

```
L_pos = (1 - s_pos)²                           # 温和拉近
L_neg = ReLU(margin - s_pos + s_hardest_neg)   # 只推开最难混淆者
```

- 没有温度参数 → 没有 softmax 的硬竞争
- margin 控制"正样本需要比负样本相似多少才算安全"
- 只关心最难区分的那个负样本（hardest negative mining）

**优势**：避免了 InfoNCE 的"赢家通吃"问题，理论上更适合密集匹配任务。

## 5. 核心发现：为什么语义特征不等于几何特征

这是本项目最深刻的发现，也是所有微调尝试失败的根本原因。

### 5.1 两种特征的本质区别

| 属性 | 语义特征（DINOv3 擅长的） | 几何特征（匹配需要的） |
|------|--------------------------|----------------------|
| 相似性规律 | "门把手 ≈ 门把手" | "这个特定的门把手左上角 ≠ 那个特定的门把手左上角" |
| 空间分布 | 平滑的语义区域 | 稀疏的、可区分的角点/边缘 |
| 实例行为 | 跨实例泛化（越多越好） | 实例内区分（越精确越好） |
| 视角行为 | 视角不变（鲁棒性） | 视角协变（精确对应） |
| 理想特征空间 | 连续的语义流形 | 离散的可区分点簇 |

### 5.2 语义平滑性：DINO 的优势，也是微调的诅咒

DINOv3 的 Zero-Shot 匹配之所以有效，是因为它的**语义平滑性**：
- 同一个物体的不同部分在特征空间中形成一个连续的流形
- 从这个物体不同视角拍摄的照片，在这个流形上投影到相近的位置
- MNN 在这条流形上找到的最近邻，大概率是正确的对应

任何形式的对比微调都在**破坏这种语义平滑性**：
- "拉近对应 patch" → 要求流形上的某两个特定点跳在一起
- "推开非对应 patch" → 要求原本在流形上相邻的点被撕开
- 连续流形被撕裂成离散点簇 → 跨视角泛化能力丧失

### 5.3 为什么"拉近正样本"本身就是错误的操作

在密集匹配中，"正样本对"——即两个视角下对应同一个 3D 点的 patch——**根本就不应该有完全相同的特征**。

原因：
1. **光照不同**：同一 3D 点在不同视角下接收到不同的光照
2. **遮挡关系不同**：一个视角可见的点在另一个视角可能部分遮挡
3. **透视形变**：16×16 的 patch 在两个视角下覆盖的实际区域不完全相同
4. **周围 context 不同**：DINO 的 attention 混合了周围 patch 的信息，不同视角的周围 context 必然不同

DINO 特征天然地 encode 了这些视角差异——这是它的**优势**，因为视角差异是位姿估计的关键信号。

**"拉近正样本" = 在教模型忽略这些差异 = 在教模型丢弃位姿估计需要的信号。**

### 5.4 思想实验：图书馆重排

想象 DINOv3 的特征空间是一个完美组织的图书馆：
- 所有"门"在一个区域，所有"窗"在另一个区域
- 每个区域内，相似的书更靠近
- Zero-Shot 匹配 = 在图 A 的"门把手"区域找一本书，在图 B 的"门把手"区域找最近的书

对比学习微调 = 告诉图书管理员：
> "把这本书和那本书放在一起（它们是同一个 3D 点），同时把这本书和其他所有看起来相似的书分开。"

图书管理员为了执行这个指令，必须：
1. 打破"门把手"区域的语义聚类
2. 把长得像的书强行分开
3. 最终整个图书馆的语义秩序被摧毁

这不是微调，是微破坏。

## 6. InfoNCE 的数学缺陷与 Mode Collapse

### 6.1 Mode Collapse 的数学证明

当模型输出完全随机的均匀特征时，所有余弦相似度独立同分布。正样本和 K 个负样本之间没有区分度，InfoNCE 损失收敛到：

```
E[L] = ln(K+1)
```

在我们的设置中，K=128 个负样本，所以：
```
E[L] = ln(129) ≈ 4.8598
```

我们在 Projection Head 实验中**精确地**观察到了这个值——证明特征空间已经完全坍塌为随机噪声。

### 6.2 τ=0.07 的致命放大效应

温度为 0.07 意味着梯度放大了 14 倍。在密集匹配中：
- 一面白墙上有 100 个语义几乎相同的 patch
- InfoNCE 强制选出"最像的那个"作为赢家
- 其他 99 个被推开——即使它们在语义上天然应该相似
- 模型为了满足这个无理的要求，只能破坏整个语义结构

### 6.3 为什么增大 batch_size 也不能彻底解决

增大 batch_size（从 1 到 8）引入了跨图像负样本（Inter-Image Negatives）。跨图像的两个"白墙"patch 来自不同场景，推开它们不那么有害。

但问题在于：**softmax 的竞争机制本身**仍然存在。即使有跨图像负样本，同一个正样本仍然需要与 K 个负样本竞争，softmax 的"赢家通吃"效应仍然在起作用。只是破坏的程度减轻了，但破坏的本质没有改变。

---

## 7. Safe Radius：空间感知的对比学习改进

### 7.1 算法动机

传统 InfoNCE 把所有非对应 patch 都视为负样本。但空间相邻的 patch 天然应该相似（它们在物理上相邻，看的是同一个表面的相邻部分）。强制推开它们等于在连续表面上制造裂缝。

### 7.2 算法设计

Safe Radius = 5 个 patch 的距离。在采样负样本时：

```
if batch_size == 1（同一图像内的负样本）:
    对每个正样本 patch，计算它与每个候选负样本的空间距离
    如果距离 < safe_radius（5 patches），将该负样本从对比中排除（mask掉）
```

批内不同图像之间的负样本（Inter-Image Negatives）永远不被 mask——不同图像的 patch 不存在空间相邻关系。

### 7.3 效果与局限

Safe Radius 将 ScanNet Precision 从 5.51% 恢复到 28.83%（Zero-Shot 为 32.20%）——证明了"不合理的负样本"确实是退化的重要原因。

**局限**：Safe Radius 只处理了**空间**相邻的负样本，没有处理**语义**相似的负样本。两面不同位置的白墙，空间上相隔甚远，但语义上几乎完全相同。它们在 InfoNCE 中仍然被互相推开，导致语义结构仍然被破坏。

---

## 8. StableMatchingLoss：去 Softmax 化的尝试

### 8.1 设计动机

如果 InfoNCE 的问题是 softmax 的硬竞争，那就不使用 softmax。StableMatchingLoss 是一个 manual design 的复合损失函数：

### 8.2 四部分损失

```
L_total = L_pos + λ_neg·L_neg + λ_div·L_div + λ_reg·L_reg
```

**L_pos（正样本吸引）**：
```
L_pos = (1 - s_pos)²
```
温和拉近正样本对。使用平方损失而非 hinge loss，因为正样本对不需要完全相同（见第 5 节）。

**L_neg（负样本排斥 — Hardest Negative Mining）**：
```
L_neg = ReLU(margin - s_pos + s_hardest_neg)
```
只推开最危险的那个混淆者。margin = 0.3 意味着：正样本只需要比最难区分的负样本高出 0.3 的相似度。

为什么只取 hardest？因为密集匹配中大多数负样本天然就有低相似度，推开它们没有信息量。只有那个与正样本相似度接近的"伪装者"才值得推开。

**L_div（多样性正则）**：
```
L_div = mean(off-diagonal of similarity_matrix)
```
同一图像内不同 patch 的特征应该保持多样性。防止所有 patch 坍缩到同一点。

**L_reg（特征保全正则）**：
```
L_reg = ||features_finetuned - features_pretrained_detached||²
```
微调后的特征不应偏离预训练特征太远。这是"信任预训练知识"的直接体现。

### 8.3 为什么还是失败了

尽管 StableMatchingLoss 修复了 InfoNCE 的数学缺陷，它的核心操作仍然是"拉近正样本、推开（最难）负样本"。如第 5 节所述，这个操作本身在密集匹配中就是错误的——不是因为损失函数的设计，而是因为**优化目标与评估目标之间存在不可逾越的鸿沟**。

训练优化的是余弦相似度，评估衡量的是位姿精度。这中间隔着 MNN → USAC → 位姿分解 → 误差计算 三个不可微的操作。模型在训练时完全不知道"提高相似度"最终如何影响位姿——也许提高了，也许有害。

## 9. Matchability Predictor：不改特征的智能筛选

### 9.1 范式转变

之前所有方法：**修改 DINO 特征空间**
路线 A：**特征完全不动，只学会"哪些 patch 值得匹配"**

```
之前：DINO(可训练) → 修改后的特征 → MNN
路线A：DINO(冻结) → Predictor → 筛选高分 patch → MNN（只在筛选后的 patch 上）
```

### 9.2 为什么能工作

DINOv3 的 MNN 匹配中，错误匹配主要来自三类 patch：

| 类型 | 示例 | 为什么会错 | Predictor 应该给的分 |
|------|------|-----------|---------------------|
| 纹理缺失 | 白墙、天空 | 几十个相似 patch，NN 随机选 → 大概率错 | 低分 |
| 重复纹理 | 地砖、天花板网格 | 语义相同但位置不同的 patch 互相干扰 | 低分 |
| 几何挑战 | 同一 3D 点，视角差异大 | 真正对应点的特征不够相似 | 不可避，但不伤害 |

通过给前两类 patch 低分并将其从匹配中排除，正确匹配的占比自然提升。

### 9.3 模型架构

```
DINOv3 1024-dim 特征 (冻结)
    ↓
Linear(1024 → 256) → BatchNorm → ReLU
    ↓
Linear(256 → 64) → BatchNorm → ReLU
    ↓
Linear(64 → 1) → Sigmoid
    ↓
可匹配度分数 ∈ [0, 1]
```

参数量：~280K（约占 backbone 的 0.09%），训练极快（< 10 分钟/epoch）。

### 9.4 零风险保证

最差情况：Predictor 输出均匀分数（所有 patch 同样对待）→ 退化为 Zero-Shot。因为 backbone 完全冻结，**不可能比 Zero-Shot 更差**。这是之前所有方法都不具备的保证。

---

## 10. 正确的技术路线

基于本项目的发现，基于 DINOv3 做更好的特征匹配，正确的方法包括：

### 10.1 冻结 Backbone + 可学习匹配头

类似 SuperGlue / LightGlue 的架构：
- DINOv3 作为不可训练的特征提取器
- 在特征之上构建注意力匹配网络（cross-attention + self-attention）
- 匹配头学习"如何匹配 DINO 特征"而非"修改 DINO 特征"
- 直接优化匹配质量，端到端可微

### 10.2 Matchability 筛选（路线 A）

冻结 backbone，只训练极小的 patch 筛选器。过滤不可靠 patch 后再做 MNN。零风险，零特征破坏。

### 10.3 Zero-Shot 的工程优化（不需要训练）

完全不训练任何东西，只在推理 pipeline 上做改进：
- SNN ratio test（第二近邻距离比替代 MNN）
- 多尺度特征融合
- USAC 参数精调
- 关键点显著性预筛选

### 10.4 端到端可微匹配

使用 LoFTR、RoMa、MASt3R 等专门为匹配设计的架构。它们在设计上直接优化几何精度，不需要"语义特征适合匹配"这个假设。

## 11. 核心概念速查表

| 概念 | 定义 | 在本项目中的角色 |
|------|------|----------------|
| ViT-L/16 | 24 层、1024-dim、16 头的 Vision Transformer | DINOv3 的 backbone 架构 |
| RoPE | 旋转位置编码，通过旋转变换编码相对位置 | DINOv3 使用 2D RoPE 编码 patch 位置 |
| LayerScale | 每子层输出乘以可学习的通道级缩放因子 | 改善深层 ViT 训练稳定性 |
| Register Tokens | 4 个可学习的全局存储令牌，吸收全局信息 | 被丢弃，只用 patch tokens 做匹配 |
| MNN | 双向最近邻匹配（A→B 且 B→A 互为最近邻） | Zero-Shot 匹配策略 |
| Essential Matrix | 3×3 矩阵编码两个相机的相对位姿 | 评估时从匹配点对估计 |
| USAC | 通用鲁棒估计框架（RANSAC 的改进版） | 从含外点的匹配对中估计 E |
| AUC@k° | 位姿误差 < k° 的累积分布面积 | 核心评估指标 |
| Precision | 满足对极几何约束的匹配点占比 | 核心评估指标 |
| LoRA | 低秩适配器：冻结 W，训练 B·A（B 零初始化） | 微调方法 |
| InfoNCE | 温度加权的 softmax 对比损失 | 失败的损失函数（导致 mode collapse） |
| τ (温度) | 控制 softmax 尖锐度的参数 | τ=0.07 导致 softmax 退化为一选一 |
| Mode Collapse | 特征退化为均匀随机噪声 | InfoNCE 导致的现象 |
| Safe Radius | 空间距离 mask，排除空间相邻的负样本 | 最有效的改进（Precision 恢复到 28.83%） |
| StableMatchingLoss | 去 softmax 的复合损失（pos + hardest neg + div + reg） | 仍未能超越 Zero-Shot |
| Matchability Predictor | 冻结 backbone + 训练 patch 筛选器 | 零风险方案，待测试 |

---

> **本项目的核心教训**：预训练视觉基础模型的强大源于其在海量数据上学习到的通用语义表示。任何试图用微量数据"修正"这个表示的操作，本质上都是在用极少的信息去覆盖极多的知识。正确的做法不是修改特征，而是在特征之上构建新的能力。
>
> "不做什么"往往比"做什么"更重要——这是本项目教给我们的最深刻的一课。
